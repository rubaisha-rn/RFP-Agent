## Goal
Implement the third runtime ADK agent — Vendor Intelligence — using THREE function-calling tools. Filter blacklisted vendors, run conflict checks, predict bid range from historical data, and rank top 5.

## Why this is the most tool-heavy agent
Three tools (`query_vendors`, `run_conflict_check`, `predict_bid_range`) with mandatory call ordering. Each call writes a `function_call` + `function_response` pair to `agent_traces`, generating the richest trace evidence in the pipeline.

## Files Created / Modified by Antigravity
1. `backend/app/agents/schemas/vendor_ranking.py` — `VendorRanked`, `BidRange`, `VendorRankingOutput` with `@field_validator` (shortlist size 1..5 with graceful truncation; non-negative evaluated count).
2. `backend/app/tools/vendor_db.py` — `query_vendors(category)` with related-category fallback (IT_services → services if no direct match), returns non-blacklisted vendors only.
3. `backend/app/tools/conflict_check.py` — `run_conflict_check(vendor_id, name, flags)`. Critical prefixes: blacklisted, criminal, fraud, sanctioned. Soft keywords: pending_litigation, previous_dispute, late_delivery, warning.
4. `backend/app/tools/bid_predictor.py` — `predict_bid_range(category, estimated_value)`. Statistics over historical avg_bid_amount; falls back to ±15% band around estimated_value when no history.
5. `backend/app/agents/prompts/vendor_intel.md` — Step-by-step prompt with mandatory tool ordering, weighted scoring formula, and strict JSON output schema.
6. `backend/app/agents/agent3_vendor_intel.py` — ADK Agent (tools=[3 tools], no output_schema), event-loop trace writing mirroring Auditor pattern, `rank_vendors(job_id, classification, compliance)` async helper, test harness chaining Classifier → Auditor → Vendor Intel.

## Antigravity's own implementation summary (verbatim from agent)
- Vendors Evaluated: 5 to 7 IT vendors typically returned from Supabase.
- Critical Conflicts: BlackBox Solutions is entirely dropped by the DB query due to being blacklisted. Vendors with `fraud` or `criminal` flags would be dropped via the `conflict_check` tool.
- Soft Flagged: Quantum IT Services is properly flagged for `pending_litigation` but maintained in the candidate pool.
- Predicted Bid Range: The `bid_predictor` relies on historical median (expect ~2.2M – 2.5M PKR for IT_services).
- Trace Count: With Agent 1 (~2 rows), Agent 2 (~6 rows), and Agent 3 (~10 rows including 3 tools × 2 rows), the expected trace row total generated by the full run comes out to about 18 rows.

## Note on Rate Limits (agent's own observation)
The test harness invokes Agent 1, Agent 2, and Agent 3 in sequence. The Gemini API free tier allows 5 GenerateContent requests/minute. Because the auditor and vendor_intel agents perform multiple prompt/tool resolution loops rapidly, the test occasionally encounters `429 RESOURCE_EXHAUSTED`. Mitigated by adding `await asyncio.sleep(15)` between agent invocations in the test harness — implementation is correct, only the back-to-back test timing exceeds free-tier throughput.
